# Ubiquitous Language

## Core concepts

| Term              | Definition                                                                                             | Aliases to avoid                                        |
| ----------------- | ------------------------------------------------------------------------------------------------------ | ------------------------------------------------------- |
| **Attachment**    | A PDF, image (`.png`, `.jpg`, `.jpeg`), or Canvas file in the vault that is eligible for transcription | File, asset, source                                     |
| **Transcript**    | The Markdown file generated from an attachment                                                         | Index file, converted file, output file, generated file |
| **Transcription** | The act of converting an attachment into a transcript                                                  | Indexing, conversion, processing                        |
| **Vault**         | The Obsidian workspace folder containing all notes and attachments                                     | Directory, project                                      |
| **Canvas**        | An Obsidian `.canvas` file representing a visual node graph                                            | Canvas file                                             |

## File structure

| Term                     | Definition                                                                                                                | Aliases to avoid                      |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------- | ------------------------------------- |
| **Autogenerated marker** | The HTML comment (`<!--auto-generate-content-below-->`) that divides the user zone from the auto zone within a transcript | Marker, divider                       |
| **User zone**            | The section of a transcript above the autogenerated marker, reserved for user-written notes                               | Notes section, header                 |
| **Auto zone**            | The section of a transcript below the autogenerated marker, written and overwritten by the plugin                         | Transcript section, generated content |
| **Output template**      | The user-configurable header structure applied to every newly created transcript                                          | Template                              |
| **Transcripts folder**   | The folder (absolute or relative) where transcripts are saved                                                             | Index folder, output folder           |

## Processing

| Term               | Definition                                                                                      | Aliases to avoid           |
| ------------------ | ----------------------------------------------------------------------------------------------- | -------------------------- |
| **Converter**      | A service responsible for transcribing a specific attachment type (PDF, PNG, JPG, JPEG, Canvas) | Indexer, parser, processor |
| **Source file**    | The original attachment being transcribed                                                       | Input file                 |
| **Orphan**         | A transcript whose source attachment no longer exists in the vault                              | Stale file, dead file      |
| **Orphan cleanup** | The process of deleting orphaned transcripts during a conversion run                            | —                          |
| **File filter**    | A glob pattern controlling which attachments are eligible for transcription                     | Glob filter, path filter   |

## UI / settings

| Term               | Definition                                                                      | Aliases to avoid             |
| ------------------ | ------------------------------------------------------------------------------- | ---------------------------- |
| **Status view**    | The sidebar panel displaying per-file transcription status                      | Status panel, indexer status |
| **Status tracker** | The in-memory object that tracks the conversion state of each file during a run | Tracker                      |
| **Run on start**   | The setting that triggers automatic transcription when Obsidian loads           | Auto-run, auto-start         |

## Relationships

- A **file** is transcribed into exactly one **transcript**.
- A **transcript** is split into a **user zone** and an **auto zone** by the **autogenerated marker**.
- A **converter** processes **attachments** of one specific type and writes **transcripts** to the **transcripts folder**.
- An **orphan** is a **transcript** with no matching **attachment** in the vault; **orphan cleanup** deletes it.
- The **file filter** determines which **attachments** are visible to any **converter**.
- The **output template** defines the initial structure of a **transcript**'s **user zone**.

## Example dialogue

> **Dev:** "If a user drops a PDF into their vault, what happens?"
> **Domain expert:** "The **converter** picks it up on the next run, calls **Gemini** with the **Gemini prompt**, and writes a **transcript** to the **transcripts folder**. The **transcript** is structured by the **output template** and split at the **autogenerated marker**."
> **Dev:** "What if the user has written notes in the **transcript** and the PDF changes?"
> **Domain expert:** "The **auto zone** is overwritten, but the **user zone** above the **autogenerated marker** is preserved exactly as-is."
> **Dev:** "And if the PDF is deleted?"
> **Domain expert:** "The **transcript** becomes an **orphan**. On the next run, **orphan cleanup** deletes it — but only if it contains the **autogenerated marker**, so user-created files are never touched."
